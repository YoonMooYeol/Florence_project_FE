## 👼 프로젝트 소개 : 누리달
--------
 **누리달**은 모든 임산부를 위한 맞춤형 AI 기반 산전 관리 플랫폼입니다. 
이 서비스는 **Vue 3**와 **Vite**를 기반으로 구축되었으며, 모던하고 사용자 친화적인 인터페이스를 제공합니다.


## 🌙 핵심 목표
------
**1. 의료 접근성 향상** 
    : AI 에이전트의 철저한 검증 과정을 통한 맞춤 의료 정보를 제공합니다. 

**2. 쉬운 임산부 지원 정책 검색** 
    : 임산부의 정부 주요 정책을 쉽고 간편하게 제공합니다.

**3. 개인화된 건강 관리** 
    : AI 기술을 활용하여 임산부 개개인의 건강 상태와 필요에 맞는 맞춤형 정보를 제공함으로써, 임산부의 건강 증진과 모성 사망률 감소에 기여합니다.


>**누리달🌙**은 “AI 기반 산전 관리 시스템”이라는 개념을 넘어 대한민국에서 태어나는 소중한 생명들을 지키고자 합니다. 



## 🕹️ 기술 스택
------
- **Vue 3**
    : 반응형 UI를 구축하는 데 유리한 프레임워크로, 컴포넌트 기반 개발을 지원하여 코드의 재사용성과 유지보수성을 높입니다.
- **Vite**
    : 빠른 빌드 및 Hot Module Replacement(HMR) 기능을 제공하여 개발 중 실시간으로 변경 사항을 반영할 수 있어 개발 효율성을 크게 향상시킵니다. 특히, Vue.js와의 호환성이 뛰어나 빠른 개발 속도를 지원합니다.
- **Pinia**
    :  Vue 3에 최적화된 상태 관리 라이브러리로, 직관적인 API와 뛰어난 TypeScript 통합을 제공하여 상태 관리의 효율성을 높입니다.
- **Vue Router**
    : Vue 애플리케이션의 내비게이션을 간편하게 처리하고, 동적 및 중첩 라우팅을 지원하여 복잡한 구조도 효율적으로 관리합니다.
- **Axios**
    : HTTP 요청을 쉽게 처리할 수 있는 라이브러리로, Promise 기반의 비동기 처리와 직관적인 API를 제공해 서버와의 데이터 통신을 간편하게 합니다.
- **Tailwind CSS**
    : 유틸리티-first CSS 프레임워크로, 클래스를 조합해 빠르고 효율적으로 스타일링할 수 있으며, 커스터마이징이 용이해 유지보수성이 뛰어납니다.
- **FullCalendar**
    : JavaScript 기반의 달력 라이브러리로, 이벤트 관리와 일정 표시를 직관적으로 구현할 수 있으며, 다양한 뷰와 커스터마이징 옵션을 제공합니다.
- **Socket.IO Client**
    : 실시간 양방향 통신을 지원하는 라이브러리로, 클라이언트와 서버 간에 빠르고 안정적인 데이터 전송을 가능하게 하여 실시간 애플리케이션 구축에 최적화됩니다.


## 📱 주요 기능
------
> **1. 로그인**  
    : 소셜 로그인, ID/PW 찾기

> **2. 회원가입** 
    : 이메일 인증

> **3. 캘린더 페이지(메인)** 
    :일정 등록, llm 요약, 태교일기

> **4. 채팅 페이지** 
    : LLM, AI 에이전트

>**5. 마이 페이지** 
    : 개인 정보 관리, 임신 정보 관리 

>**6. 검색 페이지** 
    : 검색, 팔로잉, 팔로워 기능


## 📲 시작하기
------
### (1) 필수 조건
- Node.js (v16 이상)
- npm

### (2) 설치
```bash
# 프로젝트 클론
git clone [repository-url]

# 프로젝트 디렉토리로 이동
cd Florence_project_FE-1

# 의존성 설치
npm install
```

### (3) 개발 서버 실행
```bash
npm run dev
```

## 🖥️ 프로젝트 구조
------
```
src/
├── api/          # API 관련 설정
├── assets/       # 정적 리소스
├── components/   # 재사용 가능한 컴포넌트
├── composables/  # Vue 컴포지션 API
├── config/       # 설정 파일
├── router/       # 라우터 설정
├── services/     # 서비스 로직
├── store/        # Pinia 스토어
├── utils/        # 유틸리티 함수
└── views/        # 페이지 컴포넌트
```

 ## 🥒 성과 및 회고 
 ------
 ### 잘된 점
- **AI 모델 성능 최적화 성공**  
  - AI 모델 병렬 처리와 경량화 모델을 도입해 대규모 트래픽 환경에서도 실시간 추천과 예측 구현.
  - AI 기반 추천 시스템의 처리량을 300req/sec로 최적화하며 목표를 초과 달성.

- **효율적인 협업**  
  - AI 모델 개발과 데이터 파이프라인 구축 간 명확한 역할 분담을 통해 병렬로 작업 진행.
  - GitHub Actions를 활용한 CI/CD로 모델 학습과 배포 프로세스를 자동화하여 반복 작업 시간을 단축.
  - 매일 스크럼과 AI 모델 성능 리뷰를 통해 학습 결과를 즉각적으로 피드백하며 개선.  

---

### 아쉬운 점
- **초기 AI 모델 설계 부족**  
  - AI 모델 학습 데이터의 전처리 과정에서 더티 데이터와 중복 데이터 처리가 미흡하여 초기 학습 결과에 영향.
  - 학습 및 추론 파이프라인 간 의존성 증가로 모델 배포 초기 단계에서 지연 발생.


- **시간 부족으로 일부 기능 미완성**  
  - 사용자 맞춤형 자연어 추천 시스템과 사용자 행동 데이터를 활용한 강화 학습 기능 개발이 지연.
  - 데이터 시각화 대시보드 개발을 우선순위에서 제외하여 구현하지 못함.

---

### 향후 계획
- **기술적 고도화**  
  - ONNX Runtime과 TensorFlow Lite를 활용해 클라이언트 디바이스에서 실행 가능한 초경량 AI 모델 도입. 

- **추가 기능 개발**  
  - 사용자 피드백 시스템 구축으로 AI 모델의 추천 품질을 지속적으로 개선. 
  - 데이터 분석 및 예측 대시보드 개발로 AI 추천 시스템의 성과를 시각적으로 표현하며 비즈니스 인사이트 제공.
  - AI 모델의 AutoML 기능을 확장하여 새로운 사용자 행동 패턴에 적응하는 자율 학습 시스템 구현.

- **테스트 자동화 강화**  
  - AI 모델 성능 테스트를 위한 데이터 시뮬레이션 환경 구축.
  - 기존 단위 테스트 외에 AI 모델 정확도 검증, 추론 지연 시간 측정, 부하 테스트를 추가로 적용하여 안정성을 강화.
